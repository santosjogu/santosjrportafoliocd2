¿Qué es HTML5?

Conceptos fundamentales · Historia

HTML5 (HyperText Markup Language versión 5) es el lenguaje estándar para crear y estructurar el contenido de las páginas web. No es un lenguaje de programación: no tiene variables, condiciones ni bucles. Es un lenguaje de marcado que usa etiquetas para indicarle al navegador cómo debe interpretar y mostrar cada parte del contenido.

Fue desarrollado por el W3C (World Wide Web Consortium) y publicado oficialmente en 2014. Representa un salto enorme respecto a versiones anteriores: agregó soporte nativo para audio, video, gráficos 2D/3D, geolocalización, almacenamiento local y etiquetas semánticas que antes requerían plugins externos como Flash.

💡

Analogía: Si una página web fuera una casa, HTML sería la estructura (paredes, puertas, ventanas), CSS sería la decoración (pintura, muebles) y JavaScript sería la electricidad e instalaciones (todo lo que se mueve y funciona).

HTML funciona mediante etiquetas (tags) que se escriben entre corchetes angulares: <etiqueta>. La mayoría tienen una etiqueta de apertura y una de cierre: <p>texto</p>. Algunas son auto-cerradas: <img/>, <br/>.

¿Qué necesitas para empezar?

Editores · Navegadores · Entorno

La gran ventaja de HTML es que no necesitas instalar nada especial para comenzar: solo un editor de texto y un navegador. Sin embargo, los editores de código modernos hacen el trabajo mucho más cómodo y eficiente.

El editor más popular del mundo. Gratuito, ligero y con miles de extensiones. Recomendado para principiantes.

Editores en línea: no necesitas instalar nada. Perfectos para practicar y ver resultados al instante.
En línea
  • 1
    Descarga VS Code desde code.visualstudio.com e instálalo en tu computadora. Es gratuito y funciona en Windows, Mac y Linux.
  • 2
    Instala la extensión Live Server en VS Code (busca "Live Server" en el panel de extensiones). Esta extensión recarga automáticamente tu página en el navegador cada vez que guardas un cambio.
  • 3
    Crea una carpeta de proyecto en tu escritorio, por ejemplo: mi-primera-pagina/. Abre esa carpeta en VS Code con Archivo → Abrir carpeta.
  • 4
    Crea el archivo index.html dentro de esa carpeta. El nombre index.html es especial: es la página que los servidores web muestran por defecto.
  • 5
    Haz clic derecho en el archivo y selecciona "Open with Live Server". Se abrirá tu navegador y verás el resultado en tiempo real.
⌨️

Atajo mágico en VS Code: Abre tu archivo index.html, escribe solamente ! y presiona Tab. VS Code generará automáticamente toda la estructura base de HTML5. ¡Así de fácil!

La estructura base de HTML5

Esqueleto · DOCTYPE · Head · Body

Todo documento HTML5 válido debe tener una estructura mínima obligatoria. Sin ella, el navegador intentará interpretar tu página de todas formas, pero con resultados impredecibles. Aquí está el esqueleto básico que debes usar siempre:

index.html — Estructura mínima obligatoria 
<!-- 1. Declaración DOCTYPE: le dice al navegador que es HTML5 -->
<!DOCTYPE html>

<!-- 2. Elemento raíz: envuelve todo el documento -->
<html lang="es">

  <!-- 3. HEAD: metadatos, no se muestra en la página -->
  <head>
    <meta charset="UTF-8"/>                          <!-- Codificación de caracteres -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>  <!-- Responsive -->
    <meta name="description" content="Mi página de Cultura Digital 2"/>
    <title>Cultura Digital 2 – CBTis 144</title>  <!-- Título en la pestaña -->
    <link rel="stylesheet" href="estilos.css"/>    <!-- Hoja de estilos externa -->
  </head>

  <!-- 4. BODY: todo el contenido visible de la página -->
  <body>
    <h1>¡Hola, mundo!</h1>
    <p>Mi primera página con HTML5.</p>
  </body>

</html>

Cada parte tiene su función específica. El <!DOCTYPE html> no es una etiqueta HTML sino una instrucción para el navegador. El atributo lang="es" en la etiqueta <html> es importante para la accesibilidad y los motores de búsqueda. charset="UTF-8" garantiza que tu página muestre correctamente acentos y caracteres especiales del español.

⚠️

Error común: Olvidar el <!DOCTYPE html> hace que el navegador entre en "modo quirks" (compatibilidad con navegadores viejos), lo que puede cambiar radicalmente cómo se ve tu página. Siempre debe ser la primera línea.

Etiquetas esenciales

Texto · Listas · Tablas · Enlaces · Imágenes

HTML5 tiene más de 100 etiquetas, pero con dominar las más frecuentes ya puedes construir páginas completas. Aquí están las más importantes agrupadas por categoría:

Etiqueta Nombre Para qué sirve
<h1>…<h6>EncabezadosTítulos de 6 niveles. Solo debe haber un <h1> por página (el título principal).
<p>PárrafoBloque de texto. El navegador agrega márgenes automáticamente encima y debajo.
<strong>Negrita semánticaTexto importante. A diferencia de <b>, indica importancia semántica (no solo visual).
<em>Énfasis / cursivaTexto con énfasis. Más semántico que <i>.
<a href="…">Enlace (anchor)Crea hipervínculos a otras páginas, secciones o archivos. El atributo href es obligatorio.
<img src="…" alt="…">ImagenIncrusta imágenes. src es la ruta y alt es el texto alternativo (obligatorio para accesibilidad).
<ul> / <ol>Listas<ul> lista sin orden (viñetas), <ol> lista numerada. Sus elementos son <li>.
<table>TablaEstructura de datos en filas (<tr>) y celdas (<td>). Encabezados de columna con <th>.
<div>DivisiónContenedor genérico de bloque. Agrupa elementos para aplicarles estilos CSS.
<span>Fragmento en líneaContenedor genérico en línea. Útil para aplicar estilos a partes de un texto.
<br/>Salto de líneaInserta un salto de línea. No crea un nuevo párrafo.
<hr/>Línea horizontalSeparador visual entre secciones de contenido.

<h3>Subtítulo de subsección</h3>

<!-- Párrafo con texto enriquecido -->
<p>HTML5 es el lenguaje de la <strong>Web moderna</strong>. 
Fue publicado en <em>2014</em> por el W3C.</p>

<!-- Enlace externo e interno -->
<a href="https://developer.mozilla.org" target="_blank">
  Documentación oficial MDN
</a>

<!-- Imagen con texto alternativo -->

     width="120"/>

<!-- Lista no ordenada y ordenada -->
<ul>
  <li>HTML5 – Estructura</li>
  <li>CSS3 – Estilos</li>
  <li>JavaScript – Interactividad</li>
</ul>

HTML Semántico

Estructura significativa · SEO · Accesibilidad

Una de las grandes novedades de HTML5 fue la introducción de etiquetas semánticas: etiquetas que describen el significado del contenido que contienen, no solo cómo se ve. Antes de HTML5, todo se estructuraba con <div> genéricos. Ahora existen etiquetas específicas para cada parte de una página.

🔍

¿Por qué importa? Las etiquetas semánticas ayudan a los motores de búsqueda (Google, Bing) a entender mejor tu página y mejorar tu posicionamiento. También son esenciales para la accesibilidad: los lectores de pantalla para personas con discapacidad visual las usan para navegar el contenido.

estructura-semantica.html 
<body>

  <!-- Cabecera del sitio: logo, menú de navegación -->
  <header>
    <h1>CBTis 144 – Portafolio Digital</h1>
    <nav>
      <a href="#inicio">Inicio</a>
      <a href="#tareas">Tareas</a>
    </nav>
  </header>

  <!-- Contenido principal de la página -->
  <main>

    <!-- Sección temática con título -->
    <section id="tareas">
      <h2>Mis Tareas</h2>

      <!-- Artículo: contenido independiente y reutilizable -->
      <article>
        <h3>Tarea 01 – Tim Berners-Lee</h3>
        <p>El padre de la World Wide Web...</p>
      </article>
    </section>

    <!-- Barra lateral: contenido complementario -->
    <aside>
      <h3>Recursos extra</h3>
      <p>Links y referencias útiles...</p>
    </aside>

  </main>

  <!-- Pie de página: autor, copyright, links legales -->
  <footer>
    <p>© 2025 Joaquin Santos Gumeta – CBTis 144</p>
  </footer>

</body>
Etiqueta semánticaSignificado
<header>Encabezado de la página o de una sección. Generalmente contiene el logo y la navegación principal.
<nav>Navegación principal. Agrupa los enlaces de menú del sitio.
<main>Contenido principal. Debe haber solo uno por página y no debe incluir header ni footer.
<section>Sección temática. Agrupa contenido relacionado y siempre debería tener un encabezado.
<article>Artículo independiente. Contenido que podría redistribuirse solo: una noticia, un post, una tarea.
<aside>Contenido complementario. Barras laterales, anuncios, notas al margen.
<footer>Pie de página. Autor, copyright, links de contacto, redes sociales.
<figure> / <figcaption>Imagen o ilustración con su pie de foto.
<time datetime="…">Fecha u hora con formato legible por máquinas. Mejora el SEO.
<mark>Texto resaltado como relevante en el contexto actual (como un marcador fluorescente).

Formularios en HTML5

Inputs · Validación nativa · Tipos de campo

Los formularios son la principal forma en que los usuarios envían información a un sitio web: registros, contactos, búsquedas, pagos. HTML5 introdujo muchos nuevos tipos de input con validación automática, sin necesidad de JavaScript.

formulario-registro.html 
<form action="/registrar" method="POST">

  <!-- Texto simple -->
  <label for="nombre">Nombre completo:</label>
  <input type="text" id="nombre" name="nombre"
         placeholder="Ej: Joaquin Santos" required/>

  <!-- Email con validación automática -->
  <label for="email">Correo electrónico:</label>
  <input type="email" id="email" name="email"
         placeholder="correo@cbtis144.edu.mx" required/>

  <!-- Número con rango -->
  <label for="edad">Edad:</label>
  <input type="number" id="edad" name="edad"
         min="15" max="100"/>

  <!-- Lista desplegable -->
  <label for="semestre">Semestre:</label>
  <select id="semestre" name="semestre">
    <option value="1">Primer semestre</option>
    <option value="4" selected>Cuarto semestre</option>
    <option value="6">Sexto semestre</option>
  </select>

  <!-- Área de texto grande -->
  <label for="mensaje">Comentario:</label>
  <textarea id="mensaje" name="mensaje"
             rows="4" placeholder="Escribe aquí..."></textarea>

  <!-- Botón de envío -->
  <button type="submit">Enviar registro</button>

</form>
🔐

Atributos de validación HTML5: required hace el campo obligatorio, minlength/maxlength controlan la longitud del texto, pattern acepta expresiones regulares y type="email" valida el formato de correo. Todo sin una sola línea de JavaScript.

Multimedia en HTML5

Audio · Video · Canvas · Iframe

Antes de HTML5 era necesario instalar el plugin Adobe Flash para reproducir video o audio en un navegador. HTML5 eliminó esa dependencia por completo al incluir etiquetas nativas para multimedia. Hoy YouTube, Spotify y Netflix funcionan gracias a estas etiquetas.

multimedia.html 
<!-- ① VIDEO nativo -->
<video controls autoplay muted loop
       width="720" poster="miniatura.jpg">
  <source src="video.mp4"  type="video/mp4"/>
  <source src="video.webm" type="video/webm"/>
  Tu navegador no soporta video HTML5.
</video>

<!-- ② AUDIO nativo -->
<audio controls>
  <source src="musica.mp3" type="audio/mpeg"/>
  <source src="musica.ogg" type="audio/ogg"/>
</audio>

<!-- ③ CANVAS: dibujo 2D programable con JavaScript -->
<canvas id="miCanvas" width="400" height="200"></canvas>
<script>
  const ctx = document.getElementById('miCanvas').getContext('2d');
  ctx.fillStyle = '#8b00c4';
  ctx.fillRect(20, 20, 150, 80); // x, y, ancho, alto
</script>

<!-- ④ IFRAME: incrustar otra página (ej: Google Maps) -->
<iframe
  src="https://www.youtube.com/embed/XXXXXXXXXXX"
  width="720" height="405"
  title="Video de YouTube"
  allowfullscreen
></iframe>

Buenas prácticas en HTML5

Código limpio · Accesibilidad · SEO · Rendimiento

Saber escribir HTML es la primera parte. Escribirlo bien es lo que distingue a un desarrollador profesional. Estas buenas prácticas harán tu código más legible, mantenible, accesible y posicionable:

  • ✅
    Usa siempre el DOCTYPE y el atributo lang. Son obligatorios para que tu página sea válida y accesible. <!DOCTYPE html> y <html lang="es"> van siempre en la primera y segunda línea.
  • ✅
    Escribe etiquetas en minúsculas. Aunque HTML no distingue mayúsculas de minúsculas, la convención estándar es usar minúsculas: <div> no <DIV>.
  • ✅
    Usa comillas en los atributos. Siempre pon los valores entre comillas: <img src="foto.jpg">. Evita omitirlas aunque técnicamente funcione en algunos casos.
  • ✅
    Agrega siempre el atributo alt en las imágenes. Es obligatorio para la accesibilidad. Describe el contenido de la imagen: alt="Logo de HTML5 en naranja".
  • ✅
    Usa etiquetas semánticas, no solo divs. Reemplaza <div id="nav"> por <nav>, <div id="pie"> por <footer>, etc. Mejora el SEO y la accesibilidad.
  • ✅
    Un solo <h1> por página. El <h1> es el título principal. Puede haber varios <h2>, <h3>, etc., pero el H1 solo debe aparecer una vez. Es crucial para el SEO.
  • ✅
    Separa HTML, CSS y JavaScript. El HTML va en .html, el CSS en .css y el JS en .js. Evita usar estilos en línea (style="...") y scripts directamente en el HTML.
  • ✅
    Indenta correctamente tu código. Cada nivel de anidación debe tener 2 o 4 espacios de sangría. Esto hace el código legible. VS Code tiene el atajo Alt+Shift+F para formatear automáticamente.
  • ✅
    Valida tu código HTML. Usa el validador oficial del W3C en validator.w3.org para detectar errores. Un HTML válido se renderiza de forma consistente en todos los navegadores.
  • ✅
    Optimiza las imágenes antes de subirlas. Usa formatos modernos como WebP o AVIF. Una imagen pesada puede hacer que tu página cargue lenta. Herramientas como squoosh.app te ayudan a comprimirlas sin perder calidad.
🚫

Errores comunes que debes evitar: No cerrar etiquetas como </p> o </li>, anidar etiquetas de bloque dentro de etiquetas en línea (como poner un <div> dentro de un <span>), usar tablas para hacer layouts de diseño, y copiar código sin entenderlo.

📚

Recurso oficial recomendado: La mejor documentación de HTML5 es MDN Web Docs (Mozilla Developer Network) en developer.mozilla.org/es. Está disponible en español y es mantenida por la comunidad web mundial. También puedes practicar de forma interactiva en freecodecamp.org completamente gratis.
