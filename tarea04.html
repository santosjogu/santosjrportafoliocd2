¿Qué es CSS3?

Definición · Cascada · Especificidad

CSS3 (Cascading Style Sheets, nivel 3) es el lenguaje que controla toda la apariencia visual de las páginas web: colores, tipografías, tamaños, espaciados, posiciones, animaciones y mucho más. Sin CSS, toda la Web se vería como un documento de texto sin formato.

La palabra "Cascading" (en cascada) describe cómo funcionan las reglas de estilo: si hay varias reglas que afectan al mismo elemento, el navegador aplica una jerarquía (especificidad + orden) para determinar cuál gana. CSS3 es la versión actual y agrega funcionalidades que antes solo eran posibles con imágenes o JavaScript: degradados, sombras, animaciones, variables nativas, Flexbox y Grid.

🎨

Analogía: Si HTML es la estructura de una casa (paredes, puertas), CSS es el diseño de interiores: el color de las paredes, el tamaño de los muebles, la iluminación y la decoración. El mismo HTML puede verse completamente diferente solo cambiando el CSS.

Una regla CSS tiene siempre la misma estructura: un selector que apunta al elemento HTML y un bloque de declaraciones con pares propiedad–valor:

anatomia-de-una-regla.css 
/* ── Anatomía de una regla CSS ── */

selector {           /* ← ¿A qué elemento aplica? */
  propiedad: valor;  /* ← Declaración */
  propiedad: valor;  /* ← Pueden ser múltiples */
}

/* Ejemplo real */
h1 {
  color:       #8b00c4;   /* Color del texto */
  font-size:   2.5rem;   /* Tamaño de fuente */
  text-align:  center;   /* Alineación */
}

¿Cómo vincular CSS a HTML?

3 formas · Cuál usar · Prioridades

Existen tres formas de agregar CSS a un documento HTML. Cada una tiene sus casos de uso, pero en proyectos reales siempre se prefiere la hoja de estilos externa:

formas-de-vincular.html 
<!-- ① HOJA EXTERNA (✅ recomendada para proyectos) -->
<!-- Crea un archivo estilos.css y enlázalo en el <head> -->
<link rel="stylesheet" href="estilos.css"/>

<!-- ② ESTILOS INTERNOS (⚠️ solo para páginas únicas o pruebas) -->
<!-- Dentro del <head> de tu HTML -->
<style>
  body { background: #0a0010; }
</style>

<!-- ③ ESTILOS EN LÍNEA (🚫 evita usarlos, dificultan el mantenimiento) -->
<!-- Directo en el elemento HTML -->
<p style="color: red; font-size: 18px;">Texto con estilo en línea</p>
✅

Siempre usa hoja externa: Crea un archivo estilos.css separado y enlázalo con <link>. Así puedes reutilizar los mismos estilos en múltiples páginas y el código queda organizado y fácil de mantener.

⚠️

Orden de prioridad (especificidad): El estilo en línea siempre gana sobre los internos y externos. Los internos ganan sobre los externos. Por eso los estilos en línea son difíciles de sobreescribir y se convierten en un problema en proyectos grandes.

Selectores CSS

Básicos · Combinados · Pseudo-clases · Pseudo-elementos

Los selectores son el mecanismo con el que le dices a CSS "aplica estos estilos a estos elementos". Dominar los selectores es fundamental: te permite aplicar estilos de forma precisa sin tocar el HTML.

SelectorEjemploQué selecciona
Etiquetap { }Todos los elementos <p> de la página.
Clase.tarjeta { }Todos los elementos con class="tarjeta". El más usado.
ID#menu { }El elemento único con id="menu". Alta especificidad.
Universal* { }Todos los elementos. Útil para resets globales.
Descendientenav a { }Todos los <a> que estén dentro de un <nav>.
Hijo directoul > li { }Solo los <li> que son hijos directos de <ul>.
Pseudo-clasea:hover { }El enlace cuando el cursor está encima. Hay decenas de pseudo-clases.
Pseudo-elementop::first-line { }La primera línea de cada párrafo. Crea contenido virtual.
Atributoinput[type="email"]Todos los inputs de tipo email.
Múltipleh1, h2, h3 { }Aplica el mismo estilo a varios elementos separados por coma.
selectores.css 
/* ① Selector de clase (el más usado en proyectos) */
.btn-primario {
  background:    linear-gradient(135deg, #264de4, #8b00c4);
  color:         #fff;
  padding:       12px 28px;
  border-radius: 50px;
  border:        none;
  cursor:        pointer;
}

/* ② Pseudo-clase :hover (interacción del usuario) */
.btn-primario:hover {
  transform:  translateY(-3px);
  box-shadow: 0 8px 24px rgba(41, 101, 241, 0.5);
}

/* ③ Pseudo-clase :nth-child (filas alternas de tabla) */
tr:nth-child(even) {
  background: rgba(255, 255, 255, 0.04);
}

/* ④ Pseudo-elemento ::before (contenido decorativo) */
.seccion::before {
  content:     "";
  display:     block;
  width:       60px;
  height:      3px;
  background:  linear-gradient(90deg, #264de4, #8b00c4);
  margin-bottom: 16px;
}

/* ⑤ Selector de atributo */
a[target="_blank"]::after {
  content: " ↗"; /* Indica enlace externo */
}

El Modelo de Caja (Box Model)

Margin · Border · Padding · Content

El Box Model es el concepto más importante de CSS. Todo elemento HTML es una caja compuesta por cuatro capas concéntricas. Entender cómo funcionan es la clave para controlar el espaciado y tamaño de cualquier elemento:

box-model.css 
/* ── Box Model completo de una tarjeta ── */
.tarjeta {
  /* CONTENT: tamaño del contenido interno */
  width:  320px;
  height: auto;

  /* PADDING: espacio entre el contenido y el borde */
  padding: 24px 28px;     /* arriba/abajo  izq/der */
  /* Equivalente a: */
  padding-top:    24px;
  padding-right:  28px;
  padding-bottom: 24px;
  padding-left:   28px;

  /* BORDER: borde visible alrededor del elemento */
  border:        2px solid #264de4;
  border-radius: 16px;

  /* MARGIN: espacio exterior entre esta caja y otras */
  margin: 0 auto;      /* 0 arriba/abajo, auto izq/der = centrado */
  margin-bottom: 24px;

  /* ⭐ IMPORTANTE: incluye padding y border en el width */
  box-sizing: border-box;
}

/* Reset global recomendado: aplicar box-sizing a todo */
*, *::before, *::after {
  box-sizing: border-box;
}
⭐

box-sizing: border-box es el reset más importante de CSS. Sin él, si defines width: 300px y luego agregas padding: 20px, el elemento mide 340px (¡más de lo esperado!). Con border-box, el padding y el border se incluyen dentro del width declarado. Siempre ponlo como reset global.

Colores y Tipografía

Formatos de color · Fuentes · Espaciado de texto

CSS3 ofrece múltiples formas de definir colores y control total sobre la tipografía. Saber combinarlos es lo que da identidad visual a una página.

colores-y-tipografia.css 
/* ── Formatos de color en CSS3 ── */
.ejemplos-color {
  color: red;                          /* Nombre de color */
  color: #8b00c4;                      /* Hexadecimal (el más usado) */
  color: rgb(139, 0, 196);             /* RGB: rojo, verde, azul (0-255) */
  color: rgba(139, 0, 196, 0.7);      /* RGBA: igual + transparencia (0-1) */
  color: hsl(282, 100%, 38%);         /* HSL: tono, saturación, luminosidad */
  color: hsla(282, 100%, 38%, 0.8);   /* HSLA: igual + alfa */

  /* Degradados */
  background: linear-gradient(135deg, #264de4, #8b00c4);
  background: radial-gradient(circle, #264de4, #0a0010);
}

/* ── Tipografía completa ── */
p {
  /* Fuente */
  font-family:  'Montserrat', Arial, sans-serif; /* con fallback */
  font-size:    1rem;        /* 1rem = 16px por defecto */
  font-weight:  400;         /* 100 (thin) → 900 (black) */
  font-style:   normal;      /* normal | italic | oblique */

  /* Texto */
  line-height:     1.7;       /* Interlineado (sin unidad = relativo) */
  letter-spacing:  0.05em;   /* Espaciado entre letras */
  word-spacing:    0.1em;    /* Espaciado entre palabras */
  text-align:      left;     /* left | center | right | justify */
  text-transform:  uppercase;/* none | uppercase | lowercase | capitalize */
  text-decoration: none;     /* none | underline | line-through */
  text-shadow:     0 2px 8px rgba(139,0,196,0.5);
}

/* ── Google Fonts (importar fuente externa) ── */
@import url('https://fonts.googleapis.com/css2?family=Montserrat:wght@400;700&display=swap');
/* O en el HTML: <link href="fonts.googleapis.com/..." rel="stylesheet"/> */

Flexbox

Layout en una dimensión · Alineación · Distribución

Flexbox (Flexible Box Layout) es el sistema de diseño más popular de CSS3 para organizar elementos en una sola dimensión (fila o columna). Resuelve de forma elegante problemas que antes requerían trucos complicados: centrar elementos vertical y horizontalmente, distribuir espacio entre items, hacer navbars, etc.

flexbox.css 
/* ── CONTENEDOR FLEX (el padre) ── */
.contenedor {
  display:         flex;             /* Activa Flexbox */
  flex-direction:  row;             /* row | column | row-reverse | column-reverse */
  flex-wrap:       wrap;            /* nowrap | wrap | wrap-reverse */
  justify-content: space-between;  /* Distribución en el eje principal */
  /*  flex-start | flex-end | center | space-between | space-around | space-evenly */
  align-items:     center;         /* Alineación en eje secundario */
  /*  flex-start | flex-end | center | stretch | baseline */
  gap:             20px;            /* Espacio entre items (mejor que margin) */
}

/* ── ITEMS FLEX (los hijos) ── */
.item {
  flex:       1;        /* Crecer y encoger proporcionalmente */
  flex-grow:  1;        /* Cuánto crece respecto a sus hermanos */
  flex-shrink:0;        /* 0 = no encoger nunca */
  flex-basis: 200px;   /* Tamaño inicial antes de distribuir espacio */
  align-self: flex-end;/* Sobreescribe align-items solo para este item */
  order:      2;        /* Cambia el orden visual sin modificar el HTML */
}

/* ── Ejemplo: centrar perfectamente en pantalla ── */
.centrado-perfecto {
  display:         flex;
  align-items:     center;
  justify-content: center;
  min-height:      100vh;   /* Ocupa toda la altura de la pantalla */
}

/* ── Ejemplo: navbar con logo y links ── */
header {
  display:         flex;
  justify-content: space-between;
  align-items:     center;
  padding:         0 32px;
}
🎯

Truco para memorizar: justify-content controla el eje principal (horizontal en flex-direction: row) y align-items controla el eje secundario (vertical). Para centrar algo total: display:flex; justify-content:center; align-items:center. ¡Tres líneas!

CSS Grid

Layout en dos dimensiones · Columnas · Filas · Áreas

CSS Grid es el sistema de cuadrícula nativo de CSS3 para crear layouts en dos dimensiones (filas Y columnas simultáneamente). Es ideal para la estructura general de la página. Junto con Flexbox, forma el dúo perfecto para cualquier diseño.

grid.css 
/* ── CONTENEDOR GRID (el padre) ── */
.cuadricula {
  display:               grid;
  grid-template-columns: repeat(3, 1fr);  /* 3 columnas iguales */
  grid-template-rows:    auto;
  gap:                   24px;           /* Espacio entre celdas */
  column-gap:            20px;          /* Solo columnas */
  row-gap:               28px;          /* Solo filas */
}

/* ── Columnas con tamaños mixtos ── */
.layout-pagina {
  display:               grid;
  grid-template-columns: 280px 1fr;    /* Sidebar fija + contenido flexible */
}

/* ── Grid areas con nombres (el más legible) ── */
.layout-completo {
  display:              grid;
  grid-template-columns:1fr 3fr 1fr;
  grid-template-rows:   80px 1fr 60px;
  grid-template-areas:
    "header  header  header"
    "sidebar main    aside"
    "footer  footer  footer";
  min-height: 100vh;
}
header  { grid-area: header; }
nav     { grid-area: sidebar; }
main    { grid-area: main; }
aside   { grid-area: aside; }
footer  { grid-area: footer; }

/* ── Grid responsivo automático (muy poderoso) ── */
.tarjetas {
  display:               grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap:                   24px;
  /* Crea automáticamente las columnas que quepan en pantalla */
}
💡

¿Cuándo usar Grid vs Flexbox? Usa Flexbox para componentes de una dimensión: navbars, filas de botones, cards horizontales. Usa Grid para la estructura general de la página: layout de columnas, cuadrículas de tarjetas, páginas completas. En la práctica, se usan juntos: Grid para la macro-estructura, Flexbox para el interior de cada componente.

Transiciones y Animaciones

transition · animation · @keyframes · transform

CSS3 introdujo animaciones y transiciones nativas sin JavaScript. Las transiciones animan el cambio entre dos estados (ej: hover), mientras que las animaciones con @keyframes permiten secuencias complejas con múltiples pasos.

animaciones.css 
/* ① TRANSITION: anima el cambio entre dos estados */
.boton {
  background:  #264de4;
  transform:   translateY(0);
  box-shadow:  none;

  /* propiedad  duración  función    retardo */
  transition: all 0.3s ease 0s;
  /* o múltiples propiedades: */
  transition: background 0.3s ease, transform 0.2s ease, box-shadow 0.3s ease;
}
.boton:hover {
  background: #8b00c4;
  transform:  translateY(-4px);
  box-shadow: 0 12px 30px rgba(139,0,196,0.45);
}

/* ② TRANSFORM: transformaciones 2D y 3D */
.transformaciones {
  transform: translateX(20px);           /* Mover horizontal */
  transform: translateY(-10px);         /* Mover vertical */
  transform: scale(1.05);               /* Escalar */
  transform: rotate(45deg);             /* Rotar */
  transform: skew(10deg, 0deg);         /* Sesgar */
  transform: translate(-50%, -50%);    /* Centrar absolutamente */
  /* Combinar: */
  transform: translateY(-8px) scale(1.02);
}

/* ③ @keyframes: animación con múltiples pasos */
@keyframes aparecer {
  0%   { opacity: 0; transform: translateY(30px); }
  60%  { opacity: 1; transform: translateY(-5px); }
  100% { opacity: 1; transform: translateY(0); }
}

.animado {
  /* nombre      duración  función  retardo  iteraciones  dirección   relleno */
  animation: aparecer 0.8s ease 0.2s 1 normal both;
}

/* Animación infinita (ej: loader o pulso) */
@keyframes pulso {
  0%, 100% { transform: scale(1);    opacity: 1; }
  50%      { transform: scale(1.1);  opacity: 0.7; }
}
.pulsando { animation: pulso 2s ease-in-out infinite; }

Diseño Responsivo

Media Queries · Unidades relativas · Mobile-first

El diseño responsivo hace que tu página se vea bien en cualquier dispositivo: teléfonos, tablets, laptops y monitores 4K. CSS3 lo hace posible con Media Queries y el uso de unidades relativas. El enfoque moderno es Mobile-first: diseña primero para móvil y luego expande para pantallas más grandes.

responsive.css 
/* ── Unidades relativas (siempre prefiere estas) ── */
.elemento {
  font-size: 1rem;      /* Relativo al font-size raíz (16px por defecto) */
  font-size: 1.2em;     /* Relativo al font-size del padre */
  width:     80%;       /* Porcentaje del contenedor padre */
  width:     50vw;      /* 50% del ancho de la ventana (viewport width) */
  height:    100vh;     /* 100% del alto de la ventana (viewport height) */
  padding:   2rem;      /* 2 × 16px = 32px, se ajusta si cambia la raíz */
}

/* ── Tipografía fluida con clamp() ── */
h1 {
  /* Mínimo, preferido, máximo → se adapta automáticamente */
  font-size: clamp(1.8rem, 5vw, 4rem);
}

/* ── MEDIA QUERIES (enfoque Mobile-first) ── */

/* Estilos base: para móviles pequeños (320px en adelante) */
.grid-tarjetas {
  display:               grid;
  grid-template-columns: 1fr;   /* Una columna en móvil */
  gap:                   16px;
}

/* Tablet: 600px y más */
@media (min-width: 600px) {
  .grid-tarjetas {
    grid-template-columns: repeat(2, 1fr); /* 2 columnas */
  }
  .sidebar { display: block; } /* Mostrar sidebar en tablet */
}

/* Desktop: 1024px y más */
@media (min-width: 1024px) {
  .grid-tarjetas {
    grid-template-columns: repeat(3, 1fr); /* 3 columnas */
  }
}

/* Monitor grande: 1440px y más */
@media (min-width: 1440px) {
  .contenedor { max-width: 1320px; margin: 0 auto; }
}

/* Modo oscuro automático del sistema */
@media (prefers-color-scheme: dark) {
  body { background: #0a0010; color: #fff; }
}
📱

No olvides la meta viewport: Para que las Media Queries funcionen en dispositivos móviles reales, necesitas tener en el <head> del HTML: <meta name="viewport" content="width=device-width, initial-scale=1.0"/>. Sin ella, el móvil simula una pantalla de escritorio y ignora tus Media Queries.

Variables CSS y Buenas Prácticas

Custom Properties · Organización · Código limpio

Las variables CSS (Custom Properties) son uno de los aportes más importantes de CSS moderno. Permiten definir valores reutilizables que puedes cambiar en un solo lugar y se actualizan en toda la página. Son la base de cualquier sistema de diseño o tema de colores.

variables-y-reset.css 
/* ── Variables globales en :root ── */
:root {
  /* Paleta de colores */
  --color-primario:  #264de4;
  --color-acento:    #8b00c4;
  --color-texto:     #ffffff;
  --color-fondo:     #060010;
  --color-superficie:rgba(255,255,255,0.05);

  /* Tipografía */
  --fuente-principal: 'Montserrat', sans-serif;
  --fuente-codigo:    'Fira Code', monospace;
  --tam-base:         1rem;

  /* Espaciado */
  --espacio-sm: 8px;
  --espacio-md: 16px;
  --espacio-lg: 32px;
  --espacio-xl: 64px;

  /* Bordes y sombras */
  --radio-sm:  8px;
  --radio-md:  16px;
  --radio-lg:  24px;
  --sombra:    0 8px 32px rgba(0,0,0,0.3);
  --transicion:all 0.3s ease;
}

/* ── Uso de variables ── */
.tarjeta {
  background:    var(--color-superficie);
  border-radius: var(--radio-md);
  padding:       var(--espacio-lg);
  box-shadow:    var(--sombra);
  transition:    var(--transicion);
  font-family:   var(--fuente-principal);
}

/* ── Reset CSS moderno ── */
*, *::before, *::after {
  margin:     0;
  padding:    0;
  box-sizing: border-box;
}
html { scroll-behavior: smooth; }
img  { max-width: 100%; display: block; }
a    { text-decoration: none; color: inherit; }
  • ✅
    Define variables en :root. Concentra todos tus valores de color, tipografía y espaciado en variables. Si necesitas cambiar el color principal, solo modificas una línea.
  • ✅
    Usa siempre un reset CSS. Los navegadores tienen estilos por defecto distintos entre sí. Un reset garantiza que tu página se vea igual en Chrome, Firefox, Safari y Edge.
  • ✅
    Mobile-first siempre. Escribe los estilos base para móvil y usa min-width en tus Media Queries para escalar hacia pantallas grandes.
  • ✅
    Prefiere clases sobre IDs para estilos. Los IDs tienen una especificidad muy alta que es difícil de sobreescribir. Reserva los IDs para anclas y JavaScript.
  • ✅
    Organiza tu CSS en secciones. Usa comentarios para separar: Reset, Variables, Tipografía, Layout, Componentes, Animaciones, Media Queries. Tu yo del futuro te lo agradecerá.
  • ✅
    Usa gap en lugar de margins. En Flexbox y Grid, gap es más predecible y limpio que usar margin en los elementos hijos.
  • ✅
    Evita !important. Usar !important es señal de que hay un problema de especificidad. Resuelve el problema en lugar de forzar el estilo.
  • ✅
    Usa DevTools del navegador. Presiona F12 en cualquier navegador y ve a la pestaña "Styles" para inspeccionar, modificar y experimentar con CSS en tiempo real sin tocar tu archivo.
